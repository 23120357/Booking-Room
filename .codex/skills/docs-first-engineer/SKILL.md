# Documentation First Workflow

Before writing code:

1. Read all files under /docs.
2. Read package.json.
3. Identify framework versions.
4. Identify existing project architecture.
5. Summarize requirements before implementation.

Documentation is the source of truth.

Never:
- Invent APIs.
- Invent database schemas.
- Invent requirements.
- Ignore business rules.

Always:
- Follow architecture documents.
- Follow ERD.
- Follow API specifications.
- Follow UI specifications.
- Follow business rules.

Before completing a task:
1. Verify implementation against docs.
2. Verify implementation against framework documentation.
3. List assumptions.
